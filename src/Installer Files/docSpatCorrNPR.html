Examples for Different Modules in SpatCorrNPR

In this page, we provide the syntax and examples on how to use various modules of SpatCorrNPR.

Module-1: Data Import

The following lines of code can be used to import the data. The first dataset (testData.csv) used is without the original function, m values.
data = readmatrix ('testData.csv') ;
X_1d = data (: ,1) ; % for 1 - D input
X_2d = data (: ,1:2) ; % for 2 - D input
y = data (: ,3) ; % The response
If the user wishes to use a dataset with m values, they can import the testDataWithM.csv file. In that case, the data import would be like the following:
dataWithM = readmatrix('testDataWithM.csv');
X_1d = dataWithM(:,1); % for 1-D input
X_2d = dataWithM(:,1:2); % for 2-D input
y = dataWithM(:,3); % The response
m = dataWithM(:,4); % The true function
The first three columns of both datasets are the same. The second dataset has an extra column containing the true function values, which can be used for benchmark examples.

Module-2: Bandwidth Optimization

The general syntax for bandwidth optimization is as follows:
optimal_bandwidth = bwOptimization(X, y, m, kernel, objective, h_lb, h_ub, R);
where, X is the input, y is the response, m is the true function values, kernel is the kernel function used for the regression, objective is the objective function to be minimized, h_lb is the lower bound for the bandwidth, h_ub is the upper bound for the bandwidth, and R is the true covariance matrix. The output optimal_bandwidth is the optimal solution from the bandwidth optimization (scalar for 1-D X, and a 2 × 2 symmetric matrix for 2-D X).
The bandwidth optimization for 1-D data is fairly straightforward as it is a scalar value. The following lines of codes demonstrate bandwidth optimization for the Residual Sum of Squares (RSS) objective and Epanechnikov kernel. The limits for the 1-D case are [0.01, 0.5]. The syntax for calling the optimizer is as follows:
optimal_h = bwOptimization(X_1d, y, [], 'Epanechnikov', 'Residual Sum of Squares (RSS)', 0.04, 0.5, []);
If the optimization criterion is changed to Corrected GCV (GCVc), the R matrix needs to be imported, which can be done using the following lines:
RMatrix = readmatrix('trueR.csv');
optimal_h = bwOptimization(X_1d, y, [], 'Epanechnikov', 'Corrected GCV (GCVc)', 0.01, 0.5, RMatrix);
The m values are required if the Asymptotic Squared Error (ASE) objective is chosen. We can use the following lines in that case:
optimal_h = bwOptimization(X_1d, y, m, 'Epanechnikov', 'Asymptotic Squared Error (ASE)', 0.01, 0.5, []);
The optimization for the 2-D input is similar, except the bounds are required to be 2 × 2 symmetric matrices. Below are the codes for the 2-D bandwidth optimization for Residual Sum of Squares (RSS) objective and Epanechnikov kernel:
optimal_H = bwOptimization(X_2d, y, [], 'Epanechnikov', 'Residual Sum of Squares (RSS)', [0.2, 0.001; 0.001 0.2], [0.5, 0.1; 0.1 0.5], []);
The syntax for the Corrected GCV ( GCVc) criterion for the 2-D is similar to the 1-D case, which is shown below:
optimal_H = bwOptimization(X_2d, y,[], 'Epanechnikov', 'Corrected GCV (GCVc)', [0.08, 0.001; 0.001 0.08], [0.5, 0.1; 0.1 0.5], RMatrix);
For the Asymptotic Squared Error (ASE) objective, the user can execute the following lines:
optimal_H = bwOptimization(X_2d, y, m, 'Epanechnikov', 'Asymptotic Squared Error (ASE)', [0.08, 0.001; 0.001 0.08], [0.5, 0.1; 0.1 0.5], []);

Module-3: Error Evaluation

The general syntax for error evaluation is as follows:
err = evaluate_error(X, y, m, criterion, bw, kernel);
where, X is the input, y is the response, m is the true function values, criterion is the error metric to be used, bw refers to the bandwidth (scalar for 1-D X and a 2 × 2 symmetric matrix for 2-D X), and kernel is the kernel function used for the regression. The output err is the calculated error metric.
The following two snippets calculate the PRESS values for 1-D and 2-D X, respectively. The results are shown for arbitrary bandwidth values.
press_1d = evaluate_error(X_1d, y, [], 'PRESS', 0.0824, 'Epanechnikov'); % for 1-D X
press_2d = evaluate_error(X_2d, y, [], 'PRESS', [0.25 0.1; 0.1 0.25], 'Epanechnikov'); % for 2-D X
Similarly, we can find the ASE values for the datasets. The additional input here is the m values.
ase_1d = evaluate_error(X_1d, y, m, 'ASE', 0.0824, 'Epanechnikov'); % for 1-D X
ase_2d = evaluate_error(X_2d, y, m, 'ASE', [0.25 0.1; 0.1 0.25], 'Epanechnikov'); % for 2-D X

Module-4: Plotting

Plot Raw Data: This function requires only the X and y values. The syntax for plotting the raw data is:
plotRawData(X, y)
To get a scatterplot of 1-D X, the following code is used:
plotRawData(X_1d,y) % for 1-D X
For 2-D X, the following line produces a 3-D scatterplot.
plotRawData(X_2d,y) % for 2-D X
Plot Objective Function: The general syntax for plotting objective functions is:
plotObjectiveFunction(X, y, m, bwlimit, kernel, plotObjective, R_true)
where, X is the input, y is the response, m is the true function values, bwlimit contains the limits on the bandwidth, kernel is the kernel function used for the regression, plotObjective is the objective function to be plotted, and R_true is the true covariance matrix. As mentioned before, the GUI currently does not support plotting the Corrected GCV (GCVc) function; however, the toolbox is able to do that. Similar to the Bandwidth Optimization module, the true correlation matrix needs to be imported.
The two following snippets provide the syntax for plotting the Residual Sum of Squares (RSS) function with Epanechnikov kernel for 1-D X and 2-D X, respectively.
plotObjectiveFunction(X_1d, y,[], [0.01, 5], 'Epanechnikov', 'Residual Sum of Squares (RSS)', []) % for 1-D X
plotObjectiveFunction(X_2d, y,[], [0.05 0.05; 2 2], 'Epanechnikov', 'Residual Sum of Squares (RSS)', []) % for 2-D X
The syntax for using the Corrected GCV (GCVc) with Epanechnikov kernel is as follows:
RMatrix = readmatrix('trueR.csv');
plotObjectiveFunction(X_1d, y,[], [0.01, 0.5], 'Epanechnikov', 'Corrected GCV (GCVc)', RMatrix) % for 1-D X
Or,
RMatrix = readmatrix('trueR.csv');
plotObjectiveFunction(X_2d, y,[], [0.05 0.05; 2 2], 'Epanechnikov', 'Corrected GCV (GCVc)', RMatrix) % for 2-D X
The Asymptotic Squared Error (ASE) function can be plotted in the following ways:
plotObjectiveFunction(X_1d, y, m, [0.01, 5], 'Epanechnikov', 'Asymptotic Squared Error (ASE)', []) % for 1-D X
plotObjectiveFunction(X_2d, y, m, [0.05 0.05; 2 2], 'Epanechnikov', 'Asymptotic Squared Error (ASE)', []) % for 2-D X
Plot Smoothed Function: The syntax for plotting the smoothed regression function is shown below:
plotSmoothedFunction(X, y, bw, kernel, grain)
where, X is the input, y is the response, bw is the bandwidth, kernel is the kernel function used for the regression, and grain refers to the plot resolution (optional).
Below is the syntax for plotting the smoothed function for 1-D X and 2-D X, respectively.
plotSmoothedFunction(X_1d, y, 0.0824, 'Epanechnikov', []) % for 1-D X
plotSmoothedFunction(X_2d, y, [0.233 0.058; 0.058 0.311], 'Epanechnikov', []) % for 2-D X

Module-5: Covariance Estimation

The general syntax for estimating the covariance matrix and obtaining the semivariogram plot is:
CMat = covarianceEstimation(X, y, bw, kernel, nrbins, maxdist, a0, c0, n0, numobs, model, nu, processVariance);
where, X is the input, y is the response, bw is the bandwidth, kernel is the kernel function used for the regression, nrbins is the number of bins for grouping the lag distances, maxdist is the maximum distance in the dataset, a0 is the initial range estimate, c0 is the initial sill estimate, n0 is the initial nugget estimate, numObs is the number of observations per lag distance, model represents the theoretical model to be fitted, nu represents the smoothness parameter of the Matérn model, and processVariance is the variance of the process. Except for the X, y, bw, kernel, and model, other inputs are optional. The nu parameter is required if the model chosen is Matérn. The output CMat is the covariance matrix of the residuals.
Here are some example codes:
CMat = covarianceEstimation(X_1d, y, 0.0824, 'Epanechnikov', [], [], [], [], [], [], 'Exponential', [], []); % for 1-D X
CMat = covarianceEstimation(X_2d, y, [0.233 0.058; 0.058 0.311], 'Epanechnikov', [], [], [], [], [], [], 'Exponential', [], []); % for 2-D X

Module-6: Geoplotting

The general syntax for geoPlot is:
geoPlot(X, y, bw, kernel, grain, geographicRegion, Latitude, Longitude, responseType)
where, X is the input (typically 2-D spatial data), y is the response, bw is the bandwidth, kernel is the kernel function used for the regression, and grain refers to the plot resolution (optional). The geographicRegion specifies the perimeter, Latitude and Longitude refer to the perimeter latitude and longitude of any region if geographicRegion is chosen N/A, and responseType refers to the sign of the response (Positive, Negative, or other). If the geographicRegion is Contiguous US, then Latitude and Longitude are not needed. However, for any other geographic location, the geographicRegion should be N/A, and the Latitude and the Longitude need to be provided.
The first geoPlot is for the US precipitation data. The following codes are used:
geoData = readmatrix('usPrecipitation.csv');
X = geoData(:,1:2); y = geoData(:,3);
geoPlot(X, y, [3.615 0.0026; 0.0026 2.5873], 'Epanechnikov', [], 'Contiguous US', [], [], 'Positive')
The following example uses a toy dataset from Australia, and defines the perimeter by importing the perimeter data, instead of choosing the geographicRegion. Here is the code:
geoPerimeter = readmatrix('australiaPerimeter.csv');
Lat = geoPerimeter(:,1);
Lon = geoPerimeter(:,2);
 
geoData = readmatrix('ausData.csv');
X = geoData(:,1:2); y = geoData(:,3);
geoPlot(X, y, [5.2 0.04; 0.04 7.1], 'Epanechnikov', [], 'N/A', Lat, Lon, 'other')
